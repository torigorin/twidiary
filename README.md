# twidiary.

Twitter（X）のアーカイブデータ（`tweet.js`）を、日記のように読み返すためのビューワー。

すべての処理はブラウザ内で完結します。ファイルはどこにも送信されません。

## 使い方

1. [GitHub Pages のURL](#) を開く（またはこのリポジトリの `index.html` をブラウザで直接開く）
2. Twitterアーカイブに含まれる `tweet.js` を選択、またはドラッグ&ドロップ
   - tweet.js を使わず「日記だけ始める」こともできます
3. 画像を表示したい場合は、設定 (⚙) から `tweet_media` フォルダを読み込む

## できること

- **カレンダー / タイムライン表示** — 月ごとにツイートを振り返る
- **検索** — キーワードで全期間から検索
- **メディアギャラリー** — 画像だけを一覧表示、タップで元の日記にジャンプ
- **n年前の今日** — 同じ日付の過去の記録を表示（折りたたみ可、編集・削除も可能）
- **気分マーク** — 各エントリに5段階の気分を記録
- **新規日記の追加** — tweet.js とは別に、自分で日記を書ける（画像添付可）
- **編集・削除** — ツイート・日記どちらも編集・削除可能（元に戻すことも可能）
- **バックアップ / 復元** — 手書き日記・気分マークをJSONでエクスポート・インポート
- **tweet.js の追加読み込み** — 新しいアーカイブを取り込んでも既存の手書き日記は消えない

## データの保存について

- 手書き日記・気分マーク・ツイートの編集内容は、ブラウザの `localStorage` に保存されます
- サーバーには一切送信されません。このリポジトリ／GitHub Pages は「HTMLを配信するだけの場所」です
- 定期的に **設定 → バックアップ（エクスポート）** でJSONファイルを保存しておくことを推奨します
- ブラウザやデバイスを変えると `localStorage` の内容は引き継がれません。復元は「バックアップから復元」で行います

## デプロイ（GitHub Pages）

1. このリポジトリを GitHub に push
2. Settings → Pages → Source を `main` ブランチ / `/ (root)` に設定
3. 発行されたURLをiPhoneのSafariで開き、共有 → 「ホーム画面に追加」でアプリ風に使えます
